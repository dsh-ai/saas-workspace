Стратегия привлечения, гипотезы каналов, позиционирование, JTBD, промты, вводные материалы.

## При начале работы
1. Читай `../CENTRAL_STATE.md`
2. Читай `DOMAIN_STATE.md`

## Файлы
- `DOMAIN_STATE.md` — активные гипотезы, статус каналов
- `gtm-strategy.md` — стратегия выхода на рынок, фазы роста, retention
- Остальные файлы — перенесены из dev_unilist/Маркетинг/

## Структура контента (папка Контент/)

**Принцип: SEO первично. Контент не создаётся без SEO-обоснования.**

Контент-план: `Контент/content-plan.md` — единственный источник приоритетов для написания статей.
Перед написанием любой новой статьи — проверить, есть ли она в контент-плане и на каком основании из SEO.

Каждая статья живёт в отдельной папке `Контент/<slug>/`. Внутри:
- `blog-<slug>.md` — основная версия (SEO / блог)
- Вариации: `vc-<slug>.md`, `tg-<slug>.md`, `for-rop-<slug>.md`, `for-owner-<slug>.md` и т.п.

Никогда не кладём статьи напрямую в `Контент/` — только в подпапки по одной на статью.

## Влияние на другие домены
- Новые гипотезы каналов → темы для `../content/DOMAIN_STATE.md`
- Изменение позиционирования → проверить соответствие `../dev/DOMAIN_STATE.md`
- Изменение цены (из finance) → обновить messaging
