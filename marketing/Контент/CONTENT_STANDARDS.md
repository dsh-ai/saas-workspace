# Стандарты контента: метаданные и разметка

Этот файл — единый источник правды для всех статей в папке `Контент/`. При создании или обновлении любой статьи придерживаться этих стандартов.

---

## Структура файла статьи

Каждая статья — в папке `Контент/<slug>/`, файл называется `blog-<slug>.md`.

Структура файла:

```
---                          ← frontmatter (YAML)
[поля]
---

<!-- Meta title: ... -->     ← HTML-комментарии для мета-тегов
<!-- Meta description: ... -->

# [H1 — видимый заголовок]

[контент]
```

---

## Frontmatter: обязательные поля

```yaml
---
title: "SEO-заголовок страницы для тега <title>"
h1: "Видимый H1 на странице (может отличаться от title)"
slug: url-slug-bez-blog
target_query: "основной поисковый запрос"
volume: 600
cluster: 1-klient-zavis
type: pillar
---
```

| Поле | Назначение | Пример |
|---|---|---|
| `title` | Текст тега `<title>` в браузере и поиске. До 60 символов. | `"Клиент не отвечает: что делать РОПу | Unilist"` |
| `h1` | Видимый заголовок статьи. Может быть длиннее title. | `"Клиент завис после анкеты: что делать РОПу — полный разбор"` |
| `slug` | URL-часть без `/blog/`. Только латиница, дефисы. | `klient-zavis-chto-delat-ropu` |
| `target_query` | Главный ключевой запрос кластера. Точная формулировка из Вордстат. | `"почему клиент не отвечает"` |
| `volume` | Средний объём запроса в мес. (Вордстат, последние 6 мес.) | `600` |
| `cluster` | Идентификатор кластера. | `1-klient-zavis` / `2-konversiya` / `3-kontrol` |
| `type` | Тип страницы в кластере. | `pillar` / `satellite` / `landing` |

## Meta title и Meta description

После закрывающего `---` frontmatter — два HTML-комментария:

```markdown
<!-- Meta title: Клиент не отвечает после анкеты: что делать РОПу — полный гид | Unilist -->
<!-- Meta description: Клиент получил документы и пропал? Разбираем 6 причин, алгоритм действий для РОПа и как видеть активность клиента без созвонов. -->
```

**Правила:**
- Meta title: до 60 символов, включает основной ключ, в конце `| Unilist`
- Meta description: 130–155 символов, содержит боль + обещание + конкретику. Без клише типа «узнайте больше».

---

## H1

Сразу после HTML-комментариев. Один раз на страницу. Совпадает с полем `h1` из frontmatter.

```markdown
# Клиент завис после анкеты: что делать РОПу — полный разбор
```

## Структура контента

### Иерархия заголовков
- `# H1` — один, в начале
- `## H2` — основные разделы
- `### H3` — подразделы внутри H2 (только если нужно, не для красоты)

### Горизонтальный разделитель
Используется `---` для разбивки между крупными блоками (например, между intro и первым H2, между телом статьи и FAQ, перед CTA).

### Внутренние ссылки
Формат: `[анкор](/blog/slug)` или `[анкор](/страница)`

Вставлять органично в текст, не отдельным блоком. Ссылки только на реальные страницы из текущей SEO-архитектуры.

### CTA в конце
Каждая статья заканчивается конкретным призывом. Формат:
```markdown
---

Попробуйте [название инструмента] бесплатно → [/dlya-ropa](/dlya-ropa)
```

## Правила разметки

| Элемент | Правило |
|---|---|
| **Жирный** | Только для ключевых терминов или данных, которые читатель должен заметить при сканировании. Не для украшения. |
| *Курсив* | Не используется без крайней нужды |
| Bullet-списки | Только когда пункты реально перечислимы. Без жирных заголовков + двоеточия внутри буллетов. |
| Нумерованные списки | Только для последовательных шагов, где порядок важен |
| Таблицы | Для сравнений и данных. Минимум 3 строки, иначе писать текстом |
| Тире `—` | Только когда действительно нужно интонационное выделение. Не вместо запятой |

## GEO-элементы (обязательно в каждой статье)

Подробная методика → `GEO_GUIDE.md`

Минимальный набор на каждую статью:

1. **Ключевая мысль в первых 2 абзацах** — ИИ читает начало. Главный ответ на запрос должен быть виден сразу.
2. **FAQ-блок** — минимум 3 вопроса в формате `### Вопрос` + ответ в 2–4 предложения. Ставить перед CTA.
3. **Конкретные цифры и сроки** — «3 дня без ответа», «конверсия 12%». Без расплывчатых «некоторое время» и «значительно».
4. **Структурированные H2** — каждый H2 должен отвечать на конкретный вопрос, который мог бы задать РОП.

---

## Чего НЕ делать

- Не создавать статьи без SEO-обоснования (запроса из кластера)
- Не класть файлы напрямую в `Контент/` — только в подпапки
- Не использовать bullet-пункты с форматом `**Заголовок:** текст` — это AI-паттерн
- Не добавлять эмодзи в заголовки и текст
- Не писать generic-выводы («будущее за теми, кто...»)
- Не использовать слова: additionally, ключевой, важный, подчёркивает, showcases, является свидетельством, interplay, fostering

```yaml
slug: gde-teryayutsya-sdelki-b2b
target_query: "где теряются сделки в b2b"
volume: 310
cluster: 2-konversiya
type: satellite
---
```
