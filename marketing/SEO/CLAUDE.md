# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Контекст

SEO-поддомен маркетинга Unilist. Содержит материалы, инструменты и стратегии для органического продвижения.

## При начале работы

1. Читай `../../CENTRAL_STATE.md` — общее состояние проекта
2. Читай `../DOMAIN_STATE.md` — активные маркетинговые гипотезы и каналы

## Структура (ожидаемая)

- `keywords/` — семантическое ядро, кластеры ключей
- `pages/` — SEO-страницы, landing pages, структуры
- `content-plan/` — контент-план для SEO-статей (связан с `../Контент/`)
- `technical/` — технический SEO-аудит, checklist'ы
- `competitors/` — анализ конкурентов

## SEO → Контент: обязательный процесс

**Принцип: SEO первично. Контент не пишется без SEO-обоснования.**

После каждого обновления базы ключей или кластеров — обязательно выполнить аудит:

### Шаг 1. Определить нужные материалы
Из актуальных кластеров понять, какие статьи/страницы должны существовать.

### Шаг 2. Проверить наличие в `../Контент/`
- Есть папка `../Контент/<slug>/` → материал написан
- Нет → материала нет, нужно создать

### Шаг 3. Проверить публикацию на `unilist.ru/blog`
Для каждого написанного материала: опубликован ли на блоге?
- Проверить через `WebFetch` или вручную по URL `unilist.ru/blog/<slug>`

### Шаг 4. Проверить соответствие
Если опубликован — соответствует ли статья текущим ключам и кластерной логике?
- title / h1 содержит основной ключ кластера?
- Структура статьи покрывает intent кластера?
- Нет устаревших формулировок из-за смены стратегии?

### Шаг 5. Обновить контент-план
Файл: `../Контент/content-plan.md`
- Обновить статусы по таблице аудита
- Добавить в «Запланировано» материалы, которых нет
- Перенести в «Архив» статьи, которые актуальны и опубликованы

## Связи с другими доменами

- Темы статей берутся только из SEO-кластеров (не из голых идей)
- Готовые статьи хранятся в `../Контент/<slug>/` (не в этой папке)
- Контент-план: `../Контент/content-plan.md`
- Результаты SEO-каналов влияют на `../../CENTRAL_STATE.md` (метрики трафика)

## Правила именования

Статьи — в `../Контент/<slug>/blog-<slug>.md`. В SEO-папке хранятся стратегия и аналитика, не сам контент.
