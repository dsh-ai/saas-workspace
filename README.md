# saas-workspace

Операционный мозг проекта Unilist: стратегия, финансы, маркетинг, налоги, интеграции. Код — в соседнем репозитории `../dev_unilist/`.

## Структура

| Папка | Содержит |
|---|---|
| `dev/` | Указатель на кодовую базу, инфра-план, список доменов |
| `finance/` | Юнит-экономика, цены, AI-стоимость, налоговый календарь (сводка) |
| `marketing/` | Стратегия, SEO, контент, позиционирование, лендинги, email-outreach |
| `ai-architecture/` | Архитектура AI, выбор моделей, лимиты |
| `Product/` | Продуктовые артефакты |
| `Strategy/` | Стратегические материалы |
| `tracking/` | Трекшн-карта, интервью |
| `product_process/` | Биллинг, PostHog, триггерные письма |
| `Nalog/` | Налоговая конфигурация ИП (УСН 1% Пермский край) + полный календарь 2026-2027 |
| `legal/` | Юридическая база |
| `content/` | Контент-план и статьи |
| `referral/` | Реферальная программа — дизайн, экономика, manual playbook (Phase 1 в работе с 2026-04-26) |
| `tools/` | Скрипты для внешних сервисов (см. ниже) |

## tools/

- `tools/reg-ru/` — управление доменами и DNS в Reg.ru через API. См. `tools/reg-ru/README.md`. Команды: `inventory.py`, `monitor.py`, `dns.py`, `setup-vk.py`.

Перед запуском любого скрипта — `pip install requests`, креды в `.secrets/` (gitignored), IP в whitelist соответствующего API.

## Ключевые файлы

- `CLAUDE.md` — правила работы для AI-ассистента, матрица зависимостей между доменами, инфраструктура
- `CENTRAL_STATE.md` — текущее состояние проекта (метрики, решения)
- `CHANGELOG.md` — кросс-доменные изменения
- `plan-fact.md` — план/факт (синхронизируется из Битрикс24, **вручную не править**)
