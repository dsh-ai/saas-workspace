# Фича: Голосовой ввод в форме клиента (Voice Input)

**Дата:** 2026-03-28
**Статус:** Design approved, not implemented

---

## Проблема

Основной стопор для заполнения опросных листов — нежелание клиентов вводить текст вручную. Когнитивное и временное сопротивление снижает вероятность заполнения формы.

---

## Решение

Каждое текстовое поле в публичной форме (`/form/{uuid}`) получает кнопку голосового ввода. Клиент говорит — система расшифровывает и нормализует ответ через AI.

**Доступность:** только на AI-плане (4990₽/мес).

---

## UX-поведение

Каждое поле типа `text` / `textarea` получает иконку `🎤` справа (рендерится только если `survey.owner.plan === 'ai'`).

**Флоу:**
1. Клиент нажимает `🎤` — браузер запрашивает разрешение на микрофон (один раз за сессию)
2. Кнопка переходит в состояние записи (пульсация + таймер). Клиент говорит.
3. Повторное нажатие или автостоп по паузе (~2 сек) — запись останавливается
4. Поле показывает состояние "обработка..." (spinner)
5. В поле появляется нормализованный текст
6. Клиент может отредактировать вручную или записать заново

**Состояния кнопки:** `idle` → `recording` → `processing` → `done` / `error`

**Fallback:** при ошибке микрофона или API — поле остаётся пустым, тихое сообщение "Не удалось распознать, попробуйте ещё раз".

**Ограничение:** максимум 60 секунд на одно поле (лимит синхронного режима Yandex SpeechKit).

---

## Архитектура

### Data flow

```
Браузер (MediaRecorder API)
  → [POST /api/surveys/:uuid/voice-input]
      body: { audio: Blob (webm/opus), fieldId: string }

Бэкенд (NestJS)
  → Yandex SpeechKit REST API (синхронный режим, ru-RU)
      → transcription: string
  → Anthropic Haiku
      prompt: "Ты помощник, нормализующий голосовые ответы.
               Вопрос: {fieldLabel}
               Транскрипт: {transcription}
               Верни только нормализованный ответ, без пояснений."
      → normalized: string
  → return { text: normalized }
```

### Авторизация

Эндпоинт принимает тот же `sessionToken` что использует Survey Participants — отдельной авторизации не требуется.

### Проверка плана

Бэкенд проверяет `survey.owner.plan === 'ai'` перед обработкой. Если нет — 403. На фронте иконка `🎤` не рендерится при отсутствии плана.

---

## Экономика

| Компонент | Стоимость |
|-----------|-----------|
| Yandex SpeechKit (~15 сек) | ~0.75₽ |
| Haiku нормализация | ~0.01₽ |
| **Итого за поле** | **~0.76₽** |

При полном голосовом заполнении 20 полей — ~15₽ на форму. На практике клиент использует голос на части полей.

---

## Скоуп

**Входит:**
- Кнопка `🎤` у текстовых полей (`text`, `textarea`)
- Эндпоинт `POST /api/surveys/:uuid/voice-input`
- Интеграция Yandex SpeechKit (синхронный режим)
- AI-нормализация через Anthropic Haiku
- Состояния UI: idle / recording / processing / done / error
- Проверка `plan === 'ai'` на фронте и бэкенде

**Не входит (отдельные задачи):**
- Голосовой ввод для `select`, `checkbox`, `number`
- Мультиязычность
- Аналитика использования голоса
