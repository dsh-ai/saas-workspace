# Фича: Голосовой ввод в форме клиента (Voice Input)

**Дата:** 2026-03-28
**Статус:** Design approved, not implemented

---

## Проблема

Основной стопор для заполнения опросных листов — нежелание клиентов вводить текст вручную. Когнитивное и временное сопротивление снижает вероятность заполнения формы.

---

## Решение

Каждое текстовое поле в публичной форме (`/form/{uuid}`) получает кнопку голосового ввода. Клиент говорит — браузер расшифровывает речь и вставляет текст в поле. Всё происходит на стороне клиента, без бэкенда.

**Доступность:** только на AI-плане (4990₽/мес).

---

## UX-поведение

Каждое поле типа `text` / `textarea` получает иконку `🎤` справа (рендерится только если `survey.owner.plan === 'ai'` и `SpeechRecognition` поддерживается браузером).

**Флоу:**
1. Клиент нажимает `🎤` — браузер запрашивает разрешение на микрофон (один раз за сессию)
2. Кнопка переходит в состояние записи (пульсация). Клиент говорит.
3. Повторное нажатие останавливает запись — текст вставляется в поле
4. Клиент может отредактировать вручную или записать заново

**Состояния кнопки:** `idle` → `recording` → `done` / `error`

**Fallback для Firefox** (не поддерживает Web Speech API): иконка `🎤` не показывается. Клиент заполняет вручную — без сообщений об ошибке.

**Fallback при отказе микрофона:** тихое сообщение "Не удалось распознать, попробуйте ещё раз".

---

## Архитектура

### Data flow

```
Браузер (Web Speech API — SpeechRecognition)
  → транскрипция происходит в браузере, без сетевых запросов
  → transcript: string
  → вставляется в поле напрямую
```

Бэкенд-эндпоинт не нужен. Никаких дополнительных API-интеграций.

### Проверка плана

Фронт проверяет `survey.owner.plan === 'ai'` при рендере формы. Если план не AI — иконка не рендерится.

### Поддержка браузеров

| Браузер | Поддержка |
|---------|-----------|
| Chrome desktop | ✓ |
| Chrome mobile | ✓ |
| Safari desktop | ✓ |
| Safari iOS | ✓ |
| Firefox | ✗ (graceful degradation) |

Firefox — ~5% B2B-аудитории. Эти пользователи заполняют форму вручную, кнопка `🎤` не показывается.

## Скоуп

**Входит:**
- Кнопка `🎤` у текстовых полей (`text`, `textarea`)
- Web Speech API в браузере (ru-RU)
- Состояния UI: idle / recording / done / error
- Проверка `plan === 'ai'` на фронте
- Feature detection: кнопка не рендерится если API не поддерживается

**Не входит (отдельные задачи):**
- Голосовой ввод для `select`, `checkbox`, `number`
- Поддержка Firefox (Yandex SpeechKit через бэкенд — если окажется нужно)
- AI-нормализация транскрипта
- Аналитика использования голоса
