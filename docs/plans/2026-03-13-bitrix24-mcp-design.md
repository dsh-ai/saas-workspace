# Дизайн: Bitrix24 MCP с поддержкой "Моего плана"

> Дата: 2026-03-13
> Статус: Утверждён

## Цель

Подключить Claude к Bitrix24 через MCP так, чтобы задачи, создаваемые в воркспейсе, автоматически попадали в "Мой план" в Битриксе с колонкой = домен (dev, finance, marketing, ai-architecture, content, legal). Синхронизация двусторонняя: по запросу "синхронизируй" Claude тянет актуальные статусы из Битрикса обратно в файлы воркспейса.

## Архитектура

Берём [`gunnit/bitrix24-mcp-server`](https://github.com/gunnit/bitrix24-mcp-server) как основу (TypeScript, 57 инструментов). Клонируем локально, дописываем поддержку `task.planner` API, регистрируем в Claude Code.

```
~/bitrix24-mcp/
  src/
    tools/
      tasks.ts          ← существующие (create/list/update/get)
      planner.ts        ← новые инструменты для "Моего плана"
  config.json           ← маппинг доменов → секции планировщика
  .env                  ← BITRIX24_WEBHOOK_URL
```

## Новые инструменты

| Инструмент | API Битрикса | Что делает |
|---|---|---|
| `bitrix24_add_to_my_plan` | `task.planner.addtask` | Добавляет задачу в "Мой план" в колонку по домену |
| `bitrix24_list_my_plan` | `task.planner.getlist` | Читает задачи из "Моего плана" по колонкам |
| `bitrix24_sync_workspace` | `task.planner.getlist` + fs | Обновляет `todos.md` / `plan-fact.md` по статусам из Битрикса |

## Маппинг доменов

```json
{
  "dev": "Разработка",
  "finance": "Финансы",
  "marketing": "Маркетинг",
  "ai-architecture": "AI-архитектура",
  "content": "Контент",
  "legal": "Юридическое"
}
```

Хранится в `config.json`, редактируется вручную.

## Флоу

**Создание задачи:**
1. Claude вызывает `bitrix24_create_task` → задача создана в Битриксе
2. Claude вызывает `bitrix24_add_to_my_plan(domain="dev")` → задача попадает в колонку "dev"

**Синхронизация по запросу:**
1. Пользователь: "синхронизируй Битрикс"
2. Claude вызывает `bitrix24_list_my_plan`
3. Claude обновляет статусы задач в `todos.md` / `plan-fact.md`

## Настройка

1. Создать входящий вебхук в Битриксе (Настройки → Разработчикам → Входящий вебхук)
2. Клонировать репозиторий, установить зависимости
3. Прописать webhook URL в `.env`
4. Собрать и зарегистрировать: `claude mcp add bitrix24 -- node ~/bitrix24-mcp/dist/index.js`
