name: todos_workflow
description: Как работать с файлом todos.md — он является staging-area для идей фаундера, задачи переносятся в plan-fact.md
type: feedback
---

Когда пользователь просит "проверить todos" или "посмотри todos" — читать `todos.md`, переносить все задачи в `plan-fact.md` в подходящие разделы с пометкой `[todos]`, затем сообщить что перенесено.

**Why:** todos.md — это файл идей фаундера (staging area). plan-fact.md — рабочий документ с задачами по этапам. Задачи из todos должны попадать в plan-fact, а не оставаться в todos.

**How to apply:**
1. Прочитать todos.md
2. Для каждой задачи найти подходящий раздел в plan-fact.md (или создать новый)
3. Добавить задачу с пометкой `**[todos]**` чтобы видеть что откуда пришло
4. Если задача уже есть в plan-fact — обновить её контекст, сообщить пользователю
5. Сообщить в конце что и куда перенесено
