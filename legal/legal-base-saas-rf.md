# Юридическая база SaaS-продукта в РФ

## 1. Обязательный пакет документов
*   **Публичная оферта:** Акцепт происходит в момент оплаты. Важно прописать: сервис предоставляется "как есть", ограничение ответственности суммой подписки.
*   **Политика конфиденциальности:** Соответствие ФЗ-152. Описание процессов сбора, хранения и удаления данных.
*   **Соглашение об обработке персональных данных (ПДн):** Чек-бокс при регистрации.
*   **NDA (Соглашение о неразглашении):** Для промышленников это критично. Включить пункт о том, что данные не передаются третьим лицам для обучения сторонних нейросетей.

## 2. Регуляторные требования
*   **Уведомление Роскомнадзора:** Необходимо подать уведомление о начале обработки персональных данных через Госуслуги (статус оператора ПДн).
*   **Реестр отечественного ПО:** В перспективе — подача заявки в Минцифры для освобождения от НДС (20%) и доступа к госзакупкам.

## 3. Аргументы для Службы Безопасности (СБ) заказчика
*   **Локализация:** Серверы на базе Yandex Cloud (Tier III, сертификаты ФЗ-152, ISO 27001).
*   **Изоляция:** Данные каждого клиента хранятся в логически разделенных БД.
*   **Шифрование:** Передача данных через TLS 1.3, хранение чувствительных полей — AES-256.
